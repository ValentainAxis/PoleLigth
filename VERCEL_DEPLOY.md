# 🚀 Развертывание PoleLigth на Vercel за 10 минут

Это **самый простой способ** запустить приложение в облаке. Никаких команд, никаких сложностей — всё через интерфейс.

## 📋 Что вам понадобится:

- ✅ GitHub аккаунт (у вас уже есть — ValentainAxis)
- ✅ Gmail (для входа на Vercel)
- ✅ API ключи (GEMINI_API_KEY, Firebase URL)
- ✅ 10 минут времени

---

## 🎯 Пошаговая инструкция:

### **Шаг 1: Подготовить Firebase** (2 минуты)

Если вы ещё не делали это:

1. Перейти на https://console.firebase.google.com/
2. Создать новый проект (назвать `poligth` или как угодно)
3. Включить **Realtime Database** (регион: `europe-west1`)
4. Скопировать **Database URL** (выглядит так: `https://poligth-abc123.firebaseio.com`)
5. Перейти в **⚙️ Project Settings** → **Service Accounts**
6. Нажать **"Generate New Private Key"**
7. Скачается JSON файл — **откройте его блокнотом** и скопируйте весь текст

> ⚠️ **Важно:** Сохраните этот JSON текст где-то — он понадобится на Vercel

---

### **Шаг 2: Получить Gemini API ключ** (2 минуты)

1. Перейти на https://aistudio.google.com/app/apikeys
2. Нажать **"Create API Key"**
3. Выбрать проект (если попросит)
4. Скопировать ключ (выглядит так: `AIzaSyD...`)

---

### **Шаг 3: Открыть Vercel и подключить GitHub** (3 минуты)

1. Перейти на https://vercel.com
2. Нажать **"Sign Up"** → выбрать **"Continue with GitHub"**
3. Авторизоваться в GitHub (ValentainAxis)
4. Разрешить Vercel доступ к вашим репозиториям

---

### **Шаг 4: Импортировать проект PoleLigth** (2 минуты)

1. На Vercel нажать **"Add New"** → **"Project"**
2. В списке найти **"PoleLigth"** репозиторий
3. Нажать **"Import"**

Vercel автоматически:
- ✅ Обнаружит `package.json`
- ✅ Обнаружит `vercel.json` (мы только что его создали)
- ✅ Настроит сборку

---

### **Шаг 5: Добавить переменные окружения** (2 минуты)

Откроется страница с настройками. Там будет секция **"Environment Variables"**.

Добавьте три переменные:

#### **1️⃣ GEMINI_API_KEY**
- **Name:** `GEMINI_API_KEY`
- **Value:** (вставьте ваш API ключ из Шага 2)

#### **2️⃣ FIREBASE_DATABASE_URL**
- **Name:** `FIREBASE_DATABASE_URL`
- **Value:** `https://ваш-проект.firebaseio.com` (из Firebase Console)

#### **3️⃣ FIREBASE_SERVICE_ACCOUNT_PATH**
- **Name:** `FIREBASE_SERVICE_ACCOUNT_PATH`
- **Value:** `./firebase-service-account.json`

> После каждого добавления нажимайте **"Add"** или **"Save"**

---

### **Шаг 6: Добавить Firebase Service Account JSON** (2 минуты)

Это **самая сложная часть**, но очень просто:

1. На той же странице найдите **"Environments"** или прокрутите вниз
2. Найдите секцию для добавления файлов или создайте новую переменную
3. **АЛЬТЕРНАТИВА (проще):** 
   - Создайте переменную `FIREBASE_CONFIG`
   - Вставьте туда **весь JSON** из firebase-service-account.json (скопировать всё содержимое)

Или просто скажите мне — я создам для вас файл, который не требует Firebase JSON 😊

---

### **Шаг 7: Развернуть!** (1 минута)

1. Нажать **"Deploy"** кнопку
2. Vercel начнет собирать и развертывать приложение
3. Подождите 2-3 минуты...

**Готово! 🎉**

Vercel даст вам ссылку типа: `https://poligth.vercel.app`

Откройте эту ссылку — ваше приложение живет в облаке! 🌾

Теперь это **самая простая часть**:

1. Измените код в GitHub (в любом файле)
2. Сделайте `git push` (или коммит через GitHub интерфейс)

4. Через 2-3 минуты изменения будут live! 🚀

## ❓ Если что-то не работает:

### "Build failed" или "Deployment error"

1. Откройте **Deployments** → нажмите на последний deploy
2. Смотрите **"Build Logs"** — там будет ошибка
3. Обычно это:
   - ❌ Неправильная переменная окружения
   - ❌ Не установлены зависимости (попробуйте `npm install`)
   - ❌ Опечатка в коде

### "Cannot find firebase-service-account.json"

Решение: создайте переменную `FIREBASE_CONFIG` со всем JSON содержимым

### "Приложение не открывается"

1. Проверьте URL (должен быть `https://поле-название.vercel.app`)
2. Подождите 5 минут (иногда нужно время)
3. Откройте **Deployments** и посмотрите логи
